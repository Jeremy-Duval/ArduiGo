What's inside this sample?

 

A FormDiagram.vpp project which contains one class diagram.
 

How to use the sample?
 

1. Start VP-UML and open the FormDiagram.vpp project. After that, open the vehicle class diagram.
 
 

2. Activate the Model, right-click on the class PrivateCar and select Form Diagram -> Customize... from the popup menu.
 
 

3. The Form Diagram dialog appears. You can customize the properties for the generated diagram.
The default diagram name is the name of the model you selected, you can edit the diagram name on the text field Diagram Name.
You will notice that certain options are disabled. For example, since PrivateCar does not contain any subclass, the Subclasses option is disabled. To better understand the meaning of each option, point the mouse cursor over an option, a description and a sample picture will be displayed on the right hand side of the dialog box.
And then select the options Superclasses, Navigable classes and Containers
 
 

4. Select the tab page Presentation Options, there are Attribute Options and Operation Options that you can configure. We will use the default settings and click OK to generate the diagram.
 
 

5. A new class diagram is generated.
 
 

6. Select the package vehicle and the class Vehicle in the Model Tree View. Right-click on the selection and select Form Diagram -> Hierarchical from the popup menu.
 
 

7. A new class diagram vehicle is generated. The Hierarchical formation shows the super class and subclass relationship of the selected classes.
 
 

8. Select the package vehicle and the class MotorCycle in the Model Tree View. Right-click on the selection and select Form Diagram -> Navigation from the popup menu.
 
 

9. Another new class diagram vehicleis generated. The Navigation formation shows the association relationship of the selected classes.
 

 

Result
 

Class diagrams showing different relationships between classes are generated according to the selected form diagram formation criteria.
