What's inside this sample?
 

A School.txt file which contains the problem statement of an online enrollment system.
A TexualAnalysis.vpp which is finished this sample.
 

How to use the sample?
 

1. Start VP-UML and create a textual analysis (as shown below).
 
 

2. Click the Import Text File button on the diagram toolbar.
 
 

3. Select the file School.txt and then click Open to continue.
 
 

4. The problem statement is imported.
 
 

5. Select the word "student" on the editor and drag it to the candidate class pane (located at the upper-right corner of the textual analysis window) or right-click and select Add text as Class from the popup menu.
 
 

6. A candidate class will be created for the selected word. The data dictionary table (located below the candidate class pane) displays the detailed information of the new candidate class. Let's proceed to create other candidate classes for words that are likely to be classes.
 
 

7. To create a class model from the candidate class, right-click on the candidate class and select Create Class Model from the popup menu.
 
 

8. Switch to the Model, you will see the created class student appears. Let's proceed to create classes from other candidate classes.
 
 

9. After all the classes are created, click on the Sort button in Model and select Sort by Type.
 
 

10. Select all the classes (not candidate classes) in the Model, right-click on the selection and select Form Diagram -> Navigation from the popup menu.
   

Result
 

Candidate classes are identified from the problem statement, and classes are created from the candidate classes. A class diagram is generated from the classes.
   

Tips & Trick
 

Beside creating a class, you can also create an actor, a use case, or a work flow from a candidate class. Just right-click on the candidate class and select the corresponding menu item from the popup menu to create the desired model.
