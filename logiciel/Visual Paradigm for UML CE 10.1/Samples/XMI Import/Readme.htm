What's inside this sample?
 

A XMIImport.xml file which contains one class diagram data used to be imported in this sample.
 

How to use the sample?
 

1. Start VP-UML. Click File -> Import -> XMI... on the menu bar.
 
 

2. When the Import XMI dialog box appears, select the XMIImport.xmi file (provided in this sample), then click OK to import it.
   

Result
 

After the XMI is imported, you can see the imported models in the Model , and the imported diagrams in the Diagram Navigator. The figure below shows the imported diagram Vehicle Structure.
   

Known Issue
 

1. VP-UML supports the importing of XMI of the following versions:

-XMI 2.1 for UML 1.4 (OMG)
 
 

2. If you find that only model elements are imported and there is no diagram created, it is because the imported XMI file does not contain any diagram information. You can use the Form Diagram function to generate diagrams from the imported model elements.
   

Tips & Trick
 

1. You can import an XMI file in an existing VP-UML project. Just open the VP-UML project before importing the XMI file.
2. If the XMI file does not contain any diagram information, you can use the Form Diagram function to generate diagrams for the imported model elements.
   

Feature Availability
 

2. The time required to export the XMI file depends on the size of the project. If the project is very large, it may take several minutes to export. In this case, close other programs before performing the export.
